## 3. NPC priority (works for boxes too)

1. Example #1
```bash
A   -5
B   0

flies to A
```

2. Example #2

```bash
A   2
B   -6

flies to B
```

3. Example #3
```bash
A   0
B   -20 (appears after a while in the field of view)

flies to B
```
But in this case, even if the bot is now shooting at A, which has 50% health left, and B comes into view, it will stop shooting at A and fly towards B.

If the priority difference is about 20 points lower than the others, then one of them will always be preferred, no matter who the bot kills.

4. Example #4
```bash
A   15
B   0

If A is below 50%▼ HP, the bot will continue to kill him, if HP is above 50%▲, the bot will fly towards B.
```

Here is more information about it (thanks PUNIKA):
1. NPCs that are currently attacked and have full HP gain 10 extra points.
2. Then, every 10% of the attacked NPC's missing health grants 1 additional point.
 
Example:

```bash
An attacked NPC with 50% HP gives 15 extra points.
Attacked NPC with 20% HP 18 extra points.
```
