## 3. Prioridad NPC (también funciona para cajas)

1. Ejemplo #1
```bash
A   -5
B   0

vuela a A
```

2. Ejemplo #2

```bash
A   2
B   -6

vuela a B
```

3. Ejemplo #3
```bash
A   0
B   -20 (aparece después de un tiempo en el campo de visión)

vuela a B
```
Pero en este caso, incluso si el bot ahora está disparando a A, al que le queda un 50 % de salud, y B aparece a la vista, dejará de disparar a A y volará hacia B.

Si la diferencia de prioridad es de unos 20 puntos menos que los demás, siempre se preferirá uno de ellos, sin importar a quién mate el bot.

4. Ejemplo #4
```bash
A   15
B   0

Si A está por debajo del 50%▼ HP, el bot seguirá matándolo, si HP está por encima del 50%▲, el bot volará hacia B.
```

Aquí hay más información al respecto (gracias PUNIKA):
1. Los NPC que están siendo atacados actualmente y tienen HP completo obtienen 10 puntos extra.
2. Luego, cada 10% de la salud faltante del NPC atacado otorga 1 punto adicional.
 
Ejemplo:

```bash
Un NPC atacado con 50% HP da 15 puntos extra.
NPC atacado con 20% HP 18 puntos extra.
```
