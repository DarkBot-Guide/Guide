## 1. Windows nie może znaleźć "javaw"
Możliwe przyczyny i potencjalne rozwiązania:
    1. Nie masz zainstalowanej żadnej wersji javy na swoim komputerze. Wejdź na stronę #Windows, pobierz i zainstaluj Javę 17.

    2. Po zainstalowaniu programu java, Twoja ścieżka java nie zaktualizowała się. Spróbuj ponownie zainstalować Javę 17 lub nasz runtime.

    ⚠️ Podczas instalacji runtime, pamiętaj, aby NIE zamykać wyskakujących okien Windows Command Prompt i cierpliwie czekać, aż proces instalacji zostanie zakończony. W przeciwnym razie twoja ścieżka java nigdy nie zostanie zaktualizowana!

    Jeśli to nie rozwiązało problemu, oto rozwiązanie:
        1. Krok #1.
            1. Wyszukaj Zaawansowane ustawienia systemu
            2. Kliknij na Zmienne środowiskowe...

        2. Krok #2.
            1. Przejdź do zakładki System Variables (Zmienne systemowe)
            2. Kliknij na New...
            3. Nazwa zmiennej: JAVA_HOME
            4. Wartość zmiennej: C:\Program Files \Java \Jdk1.8.0_XXX (zastąpić znaki X swoją wersją JDK. Przykład: jdk1.8.0_201) lub możesz przeglądać katalog pliku.
            5. Upewnij się, że to JDK, a nie JRE.
            6. Kliknij OK.

        3. Krok #3.
            1. Poszukaj ścieżki i kliknij na nią dwukrotnie lub zaznacz i naciśnij Edit
            2. Usuń wszystkie ścieżki Javy (np. C:\Program Files (x86)\Common Files \Oracle \Java \javapath).
            3. Kliknij na Nowy...
            4. Wpisz %JAVA_HOME%bin
            5. Kliknij OK

## 2. Unable to access jar file DarkBot.jar

Możliwe przyczyny i rozwiązania:
1. W twoim folderze z botami nie ma pliku o nazwie DarkBot.jar, na przykład dlatego, że zmieniłeś nazwę tego pliku na DarkBot_1.jar
2. Nie masz zainstalowanej żadnej wersji java na komputerze.
- Rozwiązanie takie samo jak dla błędu Windows cannot find javaw - przewiń do góry.
3. Występuje problem z twoją ścieżką java.
- Rozwiązanie takie samo jak dla błędu Windows cannot find javaw - przewiń do góry.

4. Możesz spróbować użyć RunBot.bat:

- Utwórz plik w swoim folderze Bot. Na przykład: file.properties
- Umieść swoją nazwę użytkownika i hasło w nowym pliku

![fileProperties](public/fileProperties.png)

Edytuj plik RunBot.bat do tego: START javaw -jar DarkBot.jar -start -login plik.properties

![runbot](public/runbot.png)

Nie **nie** uruchamiaj RunBot.bat z uprawnieniami administratora!

## 4. Native library (com/sun/jna/win32-x86/jnidispatch.dll) not found in resource path [...]
Powód: Na komputerze masz zainstalowaną 32-bitową wersję java (do uruchomienia bota wymagana jest wersja 64-bitowa).
Rozwiązanie: Postępuj zgodnie z poniższymi instrukcjami:
1. Kliknij menu start i wpisz appwiz.cpl
2. Zlokalizuj 32-bitową wersję java i odinstaluj ją z komputera.
3. Jeśli masz już zainstalowany nasz runtime, przejdź do kroku 4.  Jeśli nie, wejdź na stronę #Windows i pobierz go.
4. Kliknij dwukrotnie na plik RunBot.bat i sprawdź czy teraz działa.

⚠️ Jeśli wykonałeś wszystkie kroki i otrzymujesz błąd Windows cannot find javaw, spróbuj zainstalować runtime ponownie (to naprawi twoją ścieżkę java). Po instalacji spróbuj uruchomić bota.

## 5. Cannot authorize my discord account

Komunikat o błędzie: 127.0.0.1 odmówił połączenia.
Przyczyny i rozwiązania:
1. Zamykasz okienko popup z bota (ciemnoszare okienko, które wyskakuje zaraz po uruchomieniu bota) podczas procesu autoryzacji. NIE zamykaj go, ponieważ zawsze będziesz otrzymywał wspomniany błąd.
2. Przeglądarka, której aktualnie używasz nie jest w stanie załadować strony. Ustaw domyślną przeglądarkę na chrome - tam powinna nastąpić bezproblemowa autoryzacja bota.
3. Używasz bardzo agresywnego Antywirusa, który nie pozwala przejść procesu autoryzacji - spróbuj go na chwilę wyłączyć.

## 7. Nie można zalogować się do gry - odświeżanie strony logowania lub przekierowanie na stronę wylogowania tuż po kliknięciu przycisku logowania.

Przyczyna: ograniczenia bezpieczeństwa nie pozwalają na ustawienie w przeglądarce plików cookies, które są potrzebne do uzyskania dostępu do strony backpage i uruchomienia gry.

Rozwiązanie: dla każdego systemu operacyjnego Windows Server:
 1. Otwórz zakładkę "Server Manager
 2. Przejdź do Local Server
 3. Znajdź i wyłącz konfigurację IE Enhanced Security
 4. Zamknij wszystkie okna i sprawdź czy możesz się zalogować do gry.
⚠️ Pamiętaj o ponownym uruchomieniu bota, aby zastosować zmiany!

⬇️ Jeśli z jakichś powodów nie jesteś w stanie go wyłączyć, wykonaj dalsze kroki.

Rozwiązanie: dla wszystkich systemów operacyjnych:
 1. kliknij menu start, wpisz inetcpl.cpl i naciśnij enter
 2. przejdź do zabezpieczeń, kliknij zaufane witryny i dodaj *.darkorbit.com do zaufanych witryn
 3. zamknij wszystkie okna i teraz spróbuj zalogować się do gry
 ⚠️ Pamiętaj, aby zrestartować bota, aby zastosować zmiany!

Jeśli to nie pomoże i nadal będziesz miał ten sam problem, spróbuj zmniejszyć poziom zabezpieczeń dla strefy internetowej (inetcpl.cpl => security -> internet, ustaw na medium lub low)

## 8. Wybrane API nie jest w stanie się załadować

❌ Komunikat o błędzie: Wybrany interfejs API nie może zostać załadowany. Prawdopodobnie nie masz wymaganej biblioteki DLL w swoim folderze lib.
Możliwe przyczyny i rozwiązania:

1. Na komputerze masz zainstalowaną 32-bitową wersję javy - przeinstaluj Javę lub nasze Runtimes.

2. Twoja bezwzględna (pełna) ścieżka do pliku DarkBot.jar zawiera jeden lub więcej znaków narodowych (takich jak ğ, Ş ı itp.), które nie mogą być prawidłowo rozpoznane i są zamieniane na znaki zapytania.
Na przykład:
Następująca ścieżka:
Zostanie przekonwertowana na: C:users\u0026apos; c?\u0026apos; Darkbot\u0026apos;.

🔹 Tak więc przekonwertowana ścieżka wskazuje na nieprawidłową lokalizację, w której nie można znaleźć wymaganego pliku .dll i to jest przyczyną tego błędu.

Rozwiązanie: Pozbądź się wszystkich znaków narodowych ze swojej ścieżki, zmieniając nazwę wszystkich katalogów, w których one występują. Jeśli rzeczywista nazwa użytkownika na Twoim komputerze zawiera wspomniane znaki (jak w powyższym przykładzie) i nie chcesz zawracać sobie głowy zmianą nazwy, możesz po prostu przenieść folder z botem na dowolną partycję (Na przykład na dysk C:\u200) i tam go uruchomić.

3. Bot ładuje grę w nieskończoność po odświeżeniu

Rozwiązanie: Wyczyść cache i zrestartuj bota.

## 9. Failed to load config

To się dzieje, jeśli próbujesz uruchomić bota z Javą 8. Jeśli nie jesteś donatorem, wykonaj poniższe instrukcje.

1. Kliknij menu Start, wpisz appwiz.cpl i naciśnij Enter
2. Wyszukaj Java 8 i odinstaluj ją
3. Pobierz i zainstaluj nasz runtime: https://darkbot.eu/downloads/Runtimes4DarkBot.exe, a następnie ponownie uruchom bota.
