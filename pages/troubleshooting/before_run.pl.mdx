## 1. System Windows nie może znaleźć „javaw”
Możliwe przyczyny i potencjalne rozwiązania:
     1. Nie masz zainstalowanej żadnej wersji Java na swoim komputerze. Przejdź do strony #Windows, pobierz i zainstaluj Javę 17.

     2. Po zainstalowaniu programu Java ścieżka Java nie została zaktualizowana. Spróbuj ponownie zainstalować Javę 17 lub nasze środowiska wykonawcze.

     ⚠️ Podczas instalacji środowiska wykonawczego pamiętaj, aby NIE zamykać wyskakujących okienek wiersza polecenia systemu Windows i cierpliwie czekać, aż proces instalacji się zakończy. W przeciwnym razie twoja ścieżka Java nigdy się nie zaktualizuje!

     Jeśli to nie rozwiązało problemu, oto rozwiązanie:
         1. Krok 1
             1. Wyszukaj Zaawansowane ustawienia systemu
             2. Kliknij Zmienne środowiskowe...

         2. Krok 2
             1. Przejdź do Zmienne systemowe
             2. Kliknij Nowy...
             3. Nazwa zmiennej: JAVA_HOME
             4. Wartość zmiennej: C:\Program Files\Java\jdk1.8.0_XXX (zamień X na swoją wersję JDK. Przykład: jdk1.8.0_201) lub możesz wyszukać katalog pliku.
             5. Upewnij się, że jest to JDK, a nie JRE
             6. Kliknij OK

         3. Krok 3
             1. Wyszukaj ścieżkę i kliknij ją dwukrotnie lub wybierz i naciśnij przycisk Edytuj
             2. Usuń wszystkie ścieżki Java (np. C:\Program Files (x86)\Common Files\Oracle\Java\javapath)
             3. Kliknij Nowy...
             4. Napisz %JAVA_HOME%\bin
             5. Kliknij OK

## 2. Nie można uzyskać dostępu do pliku jar DarkBot.jar

Możliwe przyczyny i rozwiązania:
1. Na przykład w folderze bota nie ma pliku o nazwie DarkBot.jar, ponieważ nazwa tego pliku została zmieniona na DarkBot_1.jar
2. Nie masz zainstalowanej żadnej wersji java na komputerze / Wystąpił problem ze ścieżką java.
- To samo rozwiązanie, co w przypadku systemu Windows nie można znaleźć błędu javaw — przewiń w górę.
3. Spróbuj użyć RunBot.bat

**NIE** uruchamiaj RunBot.bat z uprawnieniami administratora!

## 3. DarkBot.jar otwiera się jako folder, archiwum lub w innej aplikacji
Możliwe rozwiązanie:
1. Kliknij prawym przyciskiem myszy plik DarkBot.jar
2. Kliknij „Otwórz za pomocą”
3. Zaznacz „Java TM” i „Zawsze używaj tej aplikacji do plików .jar”
4. Kliknij „OK”
5. Kliknij dwukrotnie plik DarkBot.jar

## 4. Nie znaleziono biblioteki natywnej (com/sun/jna/win32-x86/jnidispatch.dll) w ścieżce [...]
Powód: Masz zainstalowaną 32-bitową wersję Java na swoim komputerze (wersja 64-bitowa jest wymagana do uruchomienia bota).
Rozwiązanie: Postępuj zgodnie z poniższymi instrukcjami:
1. Kliknij menu Start i wpisz appwiz.cpl
2. Znajdź 32-bitową wersję Java i odinstaluj ją z komputera
3. Jeśli masz już zainstalowane nasze środowisko wykonawcze, przejdź do kroku 4. Jeśli nie, przejdź do strony #Windows i pobierz je.
4. Kliknij dwukrotnie plik RunBot.bat i sprawdź, czy teraz działa.

⚠️ Jeśli wykonałeś wszystkie kroki i pojawia się błąd Windows nie może znaleźć javaw, spróbuj ponownie zainstalować środowisko wykonawcze (naprawi to twoją ścieżkę java). Po instalacji spróbuj uruchomić bota.

## 5. Nie mogę autoryzować mojego konta na discordzie

Komunikat o błędzie: 127.0.0.1 odmówił połączenia.
Powody i rozwiązania:
1. Zamykasz wyskakujące okno bota (ciemnoszare okno, które pojawia się zaraz po uruchomieniu bota) podczas procesu autoryzacji. NIE zamykaj go, ponieważ zawsze pojawi się wspomniany błąd.
2. Przeglądarka, której aktualnie używasz, nie może załadować strony. Ustaw domyślną przeglądarkę na chrome - tam powinieneś bezproblemowo autoryzować bota.
3. Używasz bardzo agresywnego programu antywirusowego, który nie pozwala przejść procesu autoryzacji - spróbuj go na chwilę wyłączyć.

## 7. Brak możliwości zalogowania się do gry - odświeżenie strony logowania lub przekierowanie do strony wylogowania zaraz po kliknięciu przycisku logowania.

Przyczyna: ograniczenia bezpieczeństwa nie pozwalają na ustawienie w przeglądarce plików cookie, które są potrzebne do uzyskania dostępu do backpage'a i uruchomienia gry.

Rozwiązanie: dla dowolnego systemu operacyjnego Windows Server:
  1. Otwórz zakładkę Menedżer serwera
  2. Przejdź do opcji Serwer lokalny
  3. Znajdź i wyłącz konfigurację rozszerzonych zabezpieczeń IE
  4. Zamknij wszystkie okna i sprawdź, czy możesz zalogować się do gry.
⚠️ Pamiętaj, aby ponownie uruchomić bota, aby zastosować zmiany!

⬇️ Jeśli z jakichś powodów nie możesz go wyłączyć, postępuj zgodnie z dalszymi krokami.

Rozwiązanie: dla wszystkich systemów operacyjnych:
  1. kliknij menu start, wpisz inetcpl.cpl i naciśnij enter
  2. przejdź do bezpieczeństwa, kliknij zaufane witryny i dodaj *.darkorbit.com do zaufanych witryn
  3. zamknij wszystkie okna i teraz spróbuj zalogować się do gry
  ⚠️ Pamiętaj, aby ponownie uruchomić bota, aby zastosować zmiany!

Jeśli to nie pomoże i nadal występuje ten sam problem, spróbuj zmniejszyć poziom bezpieczeństwa dla strefy internetowej (inetcpl.cpl => bezpieczeństwo -> internet, ustaw średni lub niski).

## 8. Nie można załadować wybranego interfejsu API

❌ Komunikat o błędzie: Nie można załadować wybranego interfejsu API. Prawdopodobnie nie masz wymaganej biblioteki DLL w folderze lib.
Możliwe przyczyny i rozwiązania:

1. Masz zainstalowaną 32-bitową wersję Java na swoim komputerze - zainstaluj ponownie Javę lub nasze środowisko wykonawcze.

2. Twoja bezwzględna (pełna) ścieżka do pliku DarkBot.jar zawiera jeden lub więcej znaków narodowych (takich jak ğ, Ş ı itp.), które nie mogą zostać poprawnie rozpoznane i są zamieniane na znaki zapytania.
Na przykład:
Następująca ścieżka: C:\users\kullanıcı\Desktop\Darkbot\lib
Zostanie przekonwertowany do: C:\users\kullan?c?\Desktop\Darkbot\lib

🔹 Tak więc przekonwertowana ścieżka wskazuje nieprawidłową lokalizację, w której nie można znaleźć wymaganego pliku .dll i to jest przyczyna tego błędu.

Rozwiązanie: Pozbądź się wszystkich znaków narodowych ze swojej ścieżki, zmieniając nazwy wszystkich katalogów, w których występują. Jeśli rzeczywista nazwa użytkownika na twoim komputerze zawiera wspomniane znaki (jak w powyższym przykładzie) i nie chcesz zawracać sobie głowy zmianą jej nazwy, możesz po prostu przenieść folder bota na dowolną partycję (na przykład na dysk C:\ ) i uruchom go tam.
 
3. Bot ładuje grę w nieskończoność po odświeżeniu

Rozwiązanie: Wyczyść pamięć podręczną i uruchom ponownie bota.

## 9. Nie udało się załadować konfiguracji

Dzieje się tak, jeśli spróbujesz uruchomić bota z Javą 8. Jeśli nie jesteś dawcą, postępuj zgodnie z poniższymi instrukcjami.

1. Kliknij Menu Start, wpisz appwiz.cpl i naciśnij Enter
2. Wyszukaj Java 8 i odinstaluj ją
3. Pobierz i zainstaluj nasze środowisko wykonawcze: https://darkbot.eu/downloads/Runtimes4DarkBot.exe, a następnie ponownie uruchom bota.
